# ARCH / GARCH 波动率建模 —— 学习总结

> 配套代码：[arch_demo.py](arch_demo.py)（一条龙，均值自动选 常数 / ARMA）
> 数据/产物：`arch_returns.csv`（输入收益）、`arch_forecast.csv`（预测输出）、`arch_fig1~4_*.png`

---

## 0. 一句话理解 ARCH

金融收益率有个顽固现象：**波动会扎堆**——大涨大跌之后往往还跟着大波动，平静之后往往还是平静（**波动率聚集 volatility clustering**）。

普通模型假设方差恒定（同方差），抓不住这个。**ARCH/GARCH 就是给"方差"本身建了一个会随时间变化的模型**：用过去的冲击大小来预测今天的波动有多大。

- **ARCH 不碰均值**（一阶矩交给常数或 ARMA）；
- **ARCH 专门给方差（二阶矩）随时间的变化建模**。

---

## 1. 核心概念速查

| 概念 | 含义 |
|---|---|
| 对数收益率 $r_t=\ln(P_t/P_{t-1})$ | 把不平稳的价格转成平稳序列，是建模入口 |
| 冲击 / 残差 $a_t=r_t-\mu_t$ | 去掉均值后剩下的"意外"，ARCH 研究的对象 |
| 条件方差 $\sigma_t^2$ | 在 $t-1$ 信息下，今天波动有多大（**随时间变**） |
| 标准化残差 $\tilde a_t=a_t/\sigma_t$ | 模型对的话应是 i.i.d. 标准分布，用于检验 |
| 一阶矩 / 二阶矩 | 均值（中心） / 方差（波动） |
| 波动率聚集 | $a_t$ 不相关，但 $a_t^2$ 相关 → 用 ARCH 的理由 |
| 厚尾 | 极端值比正态多 → 新息分布用 t / 偏t / GED |

**关键认知**：$a_t$ **不相关但不独立**——线性层面（$a_t$）是白噪声，平方层面（$a_t^2$）有结构。ARCH 抓的就是藏在方差里的这种"不独立"。

### 模型方程

$$\text{ARCH}(m):\quad \sigma_t^2=\alpha_0+\alpha_1 a_{t-1}^2+\dots+\alpha_m a_{t-m}^2$$

$$\text{GARCH}(1,1):\quad \sigma_t^2=\omega+\alpha\,a_{t-1}^2+\beta\,\sigma_{t-1}^2$$

GARCH 多了 $\beta\sigma_{t-1}^2$（把"昨天的方差"也纳进来），等价于无穷阶 ARCH，能用很少参数刻画**长记忆/高持续**的波动，所以实务里 GARCH(1,1) 比高阶 ARCH 更常用。

## 2. 数据要求（喂进去之前先验收）

| 要求 | 怎么查 | 不满足怎么办 |
|---|---|---|
| **平稳** | ADF / KPSS | 价格 → 对数收益率 |
| **均值已建模** | $r_t$ 的 Ljung-Box | 有自相关 → 先上 ARMA |
| **有 ARCH 效应** | $a_t^2$ 的 Q(m) / Engle LM | 不显著 → 不用 ARCH，收工 |
| **等间隔、有序、无大缺口** | 看时间索引 | 重排、补齐或分段 |
| **样本足够**（日频≥500~1000） | 计数 | 太短不收敛 → 换更长样本 |
| **量纲适中** | 看数值大小 | ×100 转百分比 |
| **厚尾用分布兜住** | 峰度 / QQ | 选 t / 偏t / GED |
| **无突变/异常值** | 画图、滚动方差 | 复权、清洗、分段 |

> 一句话：**喂进 ARCH 的应是——等间隔、连续、足够长、已去均值结构、且残差确有波动聚集的（百分比）对数收益率序列。**

## 3. 六步建模流程（对照 demo）

```
价格 ──→ 对数收益 r_t ──→ 查均值自相关 ──→ 得冲击 a_t ──→ 查 a_t² 有无聚集
                              │(常数/ARMA)                    │(Ljung-Box/Engle LM)
                              ↓                               ↓有ARCH效应
        预测 ←── 模型检验/迭代 ←── MLE(t)估参 ←── PACF/BIC 定阶 m
     (μ+σ→区间/VaR)   (标准化残差三查)
```

### 第0步 准备数据
价格 → 对数收益率 $r_t$，ADF 验平稳。单位 ×100 转百分比。

### 第1步 均值方程（管"中心"）
**方法**：对 $r_t$ 做 Ljung-Box 检验有无自相关 → 自动分支：
- **无自相关** → $\mu_t$ 取**常数**（金融日收益常见）；
- **有自相关** → 用 **ARMA** 建 $\mu_t$。混合 ARMA 的 ACF/PACF 都拖尾、读不出阶，所以用 **BIC 网格搜索**（遍历 $p,q\in0..3$ 取 BIC 最小）定阶。

两条路都得到冲击 $a_t=r_t-\mu_t$。

> demo 里 `MEAN_KIND="const"` 走常数分支，`"arma"` 走 ARMA 分支（BIC 正确选出 ARMA(1,1)）。

### 第2步 检验 ARCH 效应（要不要建波动率模型）
对 $a_t^2$ 查波动聚集，两法互证：
- **方法1**：$a_t^2$ 的 Ljung-Box $Q(m)$；
- **方法2**：Engle LM 检验（辅助回归 $a_t^2$ 对滞后 $a_{t-i}^2$，$LM=n R^2\sim\chi^2_m$）。

p 值小 → 有 ARCH 效应，往下走；不显著 → 常数方差即可，收工。

### 第3步 定阶 m
- **主**：$a_t^2$ 的 **PACF 截尾**（ARCH(m) 类似 $a_t^2$ 的 AR(m)，PACF 在 m 阶后落入置信带）；
- **辅**：对 $m=1..M$ 各拟合，按 **AIC/BIC** 选。以 BIC 为准（更简约、相合）。

### 第4步 估计参数
- 选新息分布 $\varepsilon_t$：金融厚尾，常用 **Student-t**（也可偏 t / GED）；
- **MLE** 极大似然估出 $\omega(=\alpha_0),\alpha_1,\dots,\alpha_m,\nu$。

### 第5步 模型检验（建得够不够好）
算标准化残差 $\tilde a_t=a_t/\sigma_t$，**三查**：

| 查什么 | 工具 | 不过关 → |
|---|---|---|
| ① 均值方程 | $\tilde a_t$ 的 Ljung-Box | 改 ARMA 阶数 |
| ② 波动方程 | $\tilde a_t^2$ 的 Ljung-Box | **加大 m / 换 GARCH** |
| ③ 分布假设 | 偏度、峰度、QQ 图 | 换 t / 偏t / GED |

不过关 → 回对应步骤改，**迭代**直到 $\tilde a_t$ 干净。

> demo 的经典迭代：常数均值数据上 BIC 选了简约的 **ARCH(2)**，②检验不过（残差平方仍有结构）→ 升级 **GARCH(1,1)**，三查全过且 BIC 更低 → 采用。

### 第6步 使用模型（预测）
- **点预测**（中心）：$\mu_{t+1}$（常数 = 平线；ARMA = 逐步均值回复）；
- **波动率预测**：递推 $\sigma_t^2(\ell)$，GARCH(1,1) 为 $\sigma_t^2(\ell)=\omega+(\alpha+\beta)\sigma_t^2(\ell-1)$，随步长收敛到无条件方差 $\omega/(1-\alpha-\beta)$；
- **产出**：动态预测区间 $\mu\pm q\cdot\sigma(\ell)$（随市场松紧变宽变窄）、**VaR**（分布尾分位）。

> 见 [arch_forecast.csv](arch_forecast.csv)：每行 = 一天的 `mu_hat`(中心) + `sigma_hat`(宽度) + `lo95/hi95`(区间) + `VaR99`，`actual` 是实际值。`sigma_hat` 逐日变化 → 区间动态伸缩，这是固定方差模型做不到的。

## 4. 判据一览（每步用什么 p 值/准则决策）

| 步骤 | 判据 | 通过条件 |
|---|---|---|
| 平稳 | ADF p | < 0.05 |
| 均值要不要 ARMA | $r_t$ Ljung-Box p | < 0.05 → 上 ARMA |
| ARMA 定阶 | BIC 网格 | 取最小 |
| 有无 ARCH 效应 | $a_t^2$ Q(m) / Engle LM p | < 0.05 → 有 |
| 波动率定阶 m | $a_t^2$ PACF / AIC-BIC | 截尾阶 / 准则最小 |
| 均值检验 | $\tilde a_t$ Ljung-Box p | > 0.05 → 充分 |
| 波动检验 | $\tilde a_t^2$ Ljung-Box p | > 0.05 → 洗净 |
| 分布检验 | 偏度/峰度/QQ | 接近所选分布 |

---

## 5. 易混点澄清

1. **预测是"区间"，数据是"值"**：模型每天给 中心 $\mu_t$ + 宽度 $\sigma_t$ = 一条概率带；实际收益是带子里落下的一个点。区间在 `arch_forecast.csv` / 图里，原始收益在 `arch_returns.csv`。
2. **不相关 ≠ 独立**：$a_t$ 线性不相关，但 $a_t^2$ 相关——这正是 ARCH 存在的理由。
3. **ARCH 不预测涨跌方向**：它预测的是**波动幅度**（风险），方向由均值方程管。
4. **ARCH vs GARCH**：高持续波动用低阶 ARCH 抓不住，GARCH(1,1) 加一个 $\beta$ 项就够，更简约。
5. **工具限制**：Python `arch` 包均值只支持 AR、无 MA，完整 ARMA 要走"两步法"（statsmodels 建 ARMA 取残差 → arch 对残差建波动），完全联合估可用 R 的 `rugarch`。

| | `MEAN_KIND="const"` | `MEAN_KIND="arma"` |

| 数据均值真值 | 常数 | ARMA(1,1) |

## 7. 全流程记忆口诀

> **平稳的收益率 → 管住中心（常数/ARMA）→ 看残差波动会不会扎堆（ARCH 效应）→ 定阶、t 分布、MLE → 标准化残差三查、不行就迭代（ARCH 不够换 GARCH）→ 用 中心+波动 产出动态区间与 VaR。**
> 每一步的"门槛"都是一个假设检验的 p 值或信息准则，不达标就回上一步。

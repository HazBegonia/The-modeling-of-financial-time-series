# ARMA 模型学习总结

> 配套 demo：[arma_demo.py](arma_demo.py) —— 一条龙完整流程（造数据 → 平稳性检验 → 自动定阶 → 拟合 → 残差诊断 → 预测）。
> 本文把 ARMA 模型的核心理论和 demo 的每一步对应起来，并附上本次真实运行的数字。

---

## 1. ARMA 模型是什么

**ARMA(p,q)（自回归移动平均模型）**：把 AR 和 MA 拼在一起——当前值 = 常数 + 过去 p 个**观测值**的线性组合（AR 部分）+ 当前及过去 q 个**白噪声扰动**的线性组合（MA 部分）。

$$
y_t = c + \underbrace{\phi_1 y_{t-1} + \dots + \phi_p y_{t-p}}_{\text{AR}(p):\ 回归自己的过去值}
        + \varepsilon_t + \underbrace{\theta_1 \varepsilon_{t-1} + \dots + \theta_q \varepsilon_{t-q}}_{\text{MA}(q):\ 回归自己的过去误差}
$$

- **核心思想**：现实序列往往**既有自身惯性（AR）又受短期冲击的滞后影响（MA）**，单用 AR 或 MA 都需要很高的阶才能拟合，而 ARMA 用很少的参数就能刻画——这就是 ARMA 的**简约性（parsimony）**优势。
- **长期均值**：由 AR 部分决定，$\mu = \dfrac{c}{1-\phi_1-\dots-\phi_p}$。MA 部分均值为 0，只影响波动结构、不影响均值。
- 本 demo 真值：**ARMA(2,2)**，$y_t = 7.0 + 0.6\,y_{t-1} - 0.3\,y_{t-2} + \varepsilon_t + 0.5\,\varepsilon_{t-1} + 0.4\,\varepsilon_{t-2}$，$\sigma=1.0$，理论均值 $\mu = 7/(1-0.6+0.3)=10.0$。

### 平稳性（看 AR 部分）+ 可逆性（看 MA 部分）

ARMA 同时继承了 AR 的**平稳性**约束和 MA 的**可逆性**约束——两个部分各管一边：

- **平稳性 ← AR 部分**：AR(2) 平稳条件 $\phi_1+\phi_2<1,\ \phi_2-\phi_1<1,\ |\phi_2|<1$。demo 取 $\phi_1=0.6,\phi_2=-0.3$，三条都满足 → 平稳。
- **可逆性 ← MA 部分**：MA(2) 可逆条件 $\theta_1+\theta_2<1,\ \theta_2-\theta_1<1,\ |\theta_2|<1$。demo 取 $\theta_1=0.5,\theta_2=0.4$，三条都满足 → 可逆。

> ⚠️ **公共根（common factor）陷阱**：如果 AR 多项式和 MA 多项式有**相近的根**，它们会近似相消，使得 ARMA(2,2) 退化成更低阶的等价模型——这正是 ARMA 比 AR/MA **更难定阶**的根源（信息准则会偏向更简约的 (1,1) 等）。demo 特意把 AR 根与 MA 根选得彼此分离，避免相消，才能干净地把阶数定回 (2,2)。

## 3. 各步骤详解 + 本次运行结果

### 步骤 0 · 数据生成
用已知参数的 ARMA(2,2) 递推生成 900 个点，前 300 个为**烧入期（burn-in）丢弃**（消除初始值影响，让序列进入稳态）。留最后 `h=50` 个点做**样本外测试**，其余 850 个做训练。建模时「假装不知道阶数」，最后再揭晓真值验证。
> 注：ARMA 定阶比 AR/MA 难，需要更多数据才能让信息准则稳定区分真阶与简约近似，所以这里用了 900（AR/MA demo 用 500）。

### 步骤 1 · 平稳性检验（ADF）
**ADF 检验**：$H_0$ = 「有单位根（不平稳）」，p < 0.05 → 平稳。（检验原理——$\delta=\rho-1$ 回归、为何单位根=不平稳——见 [AR.md 步骤1](../AR_demo/AR.md)，此处不重复；ARMA 的平稳性由 AR 部分保证。）

- 本次结果：**ADF = −11.563，p = 3.24e−21 ≪ 0.05 → 平稳，可建模 ✅**
- 配图 [arma_fig1_identify.png](arma_fig1_identify.png)：序列图 + ACF + PACF。**注意看：ACF 和 PACF 都是缓慢拖尾，没有任何一个干净截尾**——这正是 ARMA 的指纹（既不是纯 AR 也不是纯 MA）。

### 步骤 2 · 定阶（本 demo：EACF + (p,q) 信息准则网格，双法交叉）

**为什么不能像 AR/MA 那样肉眼定阶？** 识别口诀：

| | ACF（自相关） | PACF（偏自相关） |
|---|---|---|
| AR(p) | 拖尾 | **p 阶后截尾** → 直接读 p |
| MA(q) | **q 阶后截尾** → 直接读 q | 拖尾 |
| **ARMA(p,q)** | **拖尾** | **拖尾** |

→ ARMA 的 ACF/PACF **双双拖尾，都不截尾**，没法从图上直接读出 p、q。所以要换**专用工具**。Tsay《金融时间序列分析》2.6.3 给了两种方法，demo 都实现并交叉印证。

> **关于搜索上限**：上限——$\text{MAXLAG}=\lfloor 12\,(T/100)^{1/4}\rfloor$（本例 $T=850\Rightarrow$ **20**，[arma_demo.py:166](arma_demo.py#L166)）。但 ARMA 的两法**成本天差地别**，所以上限的用法也不同：
> - **EACF 便宜**（只做 OLS 回归）→ **搜满整个 $[0,20]^2$ 范围**，这正是它能当主力的底气；
> - **网格贵**（每格一次 MLE）→ 高阶根本不收敛（实测 `ARIMA(10,10)` 即发散、`(20,20)` 跑 14s 仍是垃圾值），$21\times21=441$ 个拟合既慢又不可靠。**故网格只在可靠收敛的小范围 `GRIDMAX=min(MAXLAG,5)` 做交叉验证**，宽搜索交给 EACF。

**2a · EACF（扩展自相关函数）**

- **思想**（Tsay & Tiao 1984）：若能**相合地估出 AR 部分并把它滤掉**，剩下的就是一个**纯 MA 序列**，这时普通 ACF 就能看出 MA 的截尾阶。EACF 用**迭代最小二乘**近似实现这一步。
- **怎么读**：把样本 EACF 排成二维表（行 = AR 阶 $p$、列 = MA 阶 $q$）。每格与 $2/\sqrt{T}$ 比——绝对值 $\le$ 记 **O**（统计上为 0），$>$ 记 **×**（显著）。这些 **O 会聚成一个三角形，三角形的左上顶点坐标 $(p,q)$ 就是 ARMA 的阶**。
- **怎么算**（statsmodels 无内置，demo 自己实现，见 [arma_demo.py:30-104](arma_demo.py#L30-L104)）：系数按「阶 $k$ × 迭代 $j$」二维递推
  $$\hat\phi^{(k,j)}_r=\hat\phi^{(k+1,j-1)}_r-\hat\phi^{(k,j-1)}_{r-1}\cdot\frac{\hat\phi^{(k+1,j-1)}_{k+1}}{\hat\phi^{(k,j-1)}_{k}}\quad(\text{基例 }\hat\phi^{(k,0)}=\text{AR}(k)\text{ 的 OLS 系数})$$
  EACF$(k,q)$ = 用 $\hat\phi^{(k,q+1)}$ 对 $z$ 做 AR($k$) 滤波后，残差的 lag $(q+1)$ 样本自相关。
- 本次 EACF 表（O=不显著、×=显著）：

```
       q0 q1 q2 q3 q4 q5 q6 q7 q8   （完整搜到 q20，此处截左上窗口）
  p=0   x  x  o  x  x  o  o  o  o
  p=1   x  x  x  x  x  o  o  o  o
  p=2   x  x  o  o  o  o  o  o  o   ← 从这一行 q2 起连成 O
  p=3   x  x  o  o  o  x  o  o  o
  p=4   x  x  x  o  o  x  o  o  o
  p=5   x  x  x  x  x  x  o  o  o
  p=6   x  x  x  x  x  x  o  o  o
```

→ **O 三角左上顶点落在 (p,q) = (2,2)**。（高阶处几个零星 × 是有限样本噪声——Tsay 原文也提醒实际 EACF 表常偏「脏」、需结合判断；demo 用「按 $p+q$ 最小、三角左边沿 ≥80% 为 O」自动定顶点，且顶点是在完整 $[0,20]^2$ 上找、不受显示窗口影响。）

**2b · (p,q) 二维 AIC/BIC 网格** —— 对 `p=0..5, q=0..5`（网格上限 `GRIDMAX=5`，见上）每个组合拟合 `ARIMA(p,0,q)`，记录 AIC/BIC，取最小。
- **AIC/BIC 是什么**：都是「$-2\ln L+\text{复杂度罚}$」的打分，越小越好——似然 $L$ 大（拟合好）⊕ 罚参数多（防过拟合）。$\text{AIC}=-2\ln L+2k$、$\text{BIC}=-2\ln L+k\ln n$（ARMA 的 $k=p+q+2$，含常数与 $\sigma^2$）；$n>7$ 起 BIC 罚得更重、更偏小阶（详见 [AR.md 2b](../AR_demo/AR.md)）。
- ⚠️ 工程要点：① 所有组合在**同一训练样本**上拟合，IC 才可比；② 个别组合可能不收敛，用 `try/except` 跳过留 `NaN`；③ 并列时用**最简约（p+q 最小）**打破。代价：每格一次 MLE（36 个组合），且高阶不收敛——这正是网格只敢搜到 5、宽搜索交给 EACF 的原因。
- 本次 BIC 网格（行 p、列 q，越小越好）：

```
       q=0       q=1       q=2       q=3       q=4       q=5
  p=0    --     2779.6    2500.9    2439.7    2446.4    2449.5
  p=1  2747.9   2542.8    2456.4    2446.4    2448.4    2452.1
  p=2  2453.9   2457.0   *2437.7*   2444.4    2451.0    2457.1
  p=3  2455.0   2466.6    2444.4    2450.8    2456.1    2464.5
  p=4  2445.4   2448.7    2451.0    2457.4    2463.4    2468.8
  p=5  2446.3   2452.5    2456.5    2461.0    2468.4    2466.4
```

- **AIC 最小 @ (2,2)（AIC=2409.18），BIC 最小 @ (2,2)（BIC=2437.65）**。
- 配图 [arma_fig2_order.png](arma_fig2_order.png)：**左 = EACF 的 O/× 表**（搜到 Schwert=20、图示左上窗口，蓝圈标三角顶点），**右 = BIC 热力图**（红圈标最小点）——两法都指向 (2,2)。

**投票汇总**：`EACF=(2,2) | AIC=(2,2) | BIC=(2,2)` → **三法一致，最终定阶 (p,q) = (2,2)（以 BIC 为准）**。

### 步骤 3 · 拟合
**怎么拟合的（同 MA，因含 MA 部分）**：AR 部分的回归元 $y_{t-1},y_{t-2}$ 可观测，但 MA 部分的回归元 $\varepsilon_{t-1},\varepsilon_{t-2}$ **不可观测**——只要模型带 MA，就没法像纯 AR 那样直接 OLS。于是用**极大似然 + 状态空间（卡尔曼滤波）**：把 ARMA(2,2) 写成状态空间形式，卡尔曼滤波在给定参数下递推出每步的 $\hat\varepsilon_t$ 与似然，优化器再迭代调 $(\mu,\phi_1,\phi_2,\theta_1,\theta_2,\sigma^2)$ 让似然最大（[arma_demo.py:268](arma_demo.py#L268)）。这也是 ARMA/MA 比 AR 慢的原因。

用定出的 (2,2) 拟合，估计参数 vs 真值：

| 参数 | 估计值 | 真值 |
|---|---|---|
| const (μ) | +9.968 | 10.0 |
| φ₁ (ar.L1) | +0.738 | 0.6 |
| φ₂ (ar.L2) | −0.350 | −0.3 |
| θ₁ (ma.L1) | +0.447 | 0.5 |
| θ₂ (ma.L2) | +0.333 | 0.4 |
| σ² (sigma2) | +0.981 | 1.0 |

估计值都在真值附近 → 模型抓对了结构。
> 注：`statsmodels` 输出的 `const` 是**均值 μ 的估计**（不是截距 c）。

### 步骤 4 · 残差诊断（判断模型是否充分）
**好模型的残差应近似白噪声**（AR + MA 的结构都已榨干，剩下无规律可挖）。

- **Ljung–Box 检验**：$H_0$ = 「残差无自相关」。p > 0.05 → 残差是白噪声 ✅。
  本次：**Ljung-Box(10) p ≈ 1.0 ≫ 0.05 → 残差无自相关，模型充分**。
- 残差均值 ≈ 0.000，标准差 ≈ 0.991（贴近真值 σ=1.0）。
- 配图 [arma_fig3_residuals.png](arma_fig3_residuals.png)：残差时序图（无趋势）、残差 ACF（全在带内）、直方图（近正态）、Q-Q 图（贴直线）。

### 步骤 5 · 样本外预测 + 评估
对未参与训练的最后 50 个点预测并评估：

- **RMSE = 2.033，MAE = 1.642**
- **朴素基准（last-value）RMSE = 2.825** → **ARMA 更优 ✅**
- 配图 [arma_fig4_forecast.png](arma_fig4_forecast.png)：ARMA 的多步预测 = **MA 短期记忆 + AR 均值回复的叠加**——前 q=2 步 MA 项还带信息，之后由 AR 部分**平滑收敛到长期均值 μ=10**，置信区间随步数变宽。形态介于「MA 超过 q 步立刻变平」和「纯 AR 平滑回复」之间。

## 4. 关键概念速查（AR ↔ MA ↔ ARMA 三方对照）

| 维度 | AR(p) | MA(q) | **ARMA(p,q)** |
|---|---|---|---|
| 回归对象 | 过去**观测值** $y_{t-k}$ | 过去**误差** $\varepsilon_{t-k}$ | **两者都要** |
| ACF | 拖尾 | **q 阶截尾** | **拖尾** |
| PACF | **p 阶截尾** | 拖尾 | **拖尾** |
| 定阶主力 | PACF 截尾 | ACF 截尾 | **EACF 三角顶点 + (p,q) AIC/BIC 网格** |
| 平稳性 | 需 AR 条件 | 恒平稳 | **需 AR 条件**（看 AR 部分） |
| 可逆性 | 不涉及 | 需 MA 条件 | **需 MA 条件**（看 MA 部分） |
| 估计方法 | 最小二乘（快） | 极大似然（慢） | 极大似然（慢） |
| 多步预测 | 平滑收敛到 μ | 超 q 步立刻变平到 μ | **前 q 步含 MA 信息，之后 AR 平滑回复到 μ** |
| 简约性 | — | — | **小 p+小 q 即可替代高阶 AR 或 MA** |

- **ACF/PACF 都拖尾 → 认 ARMA**；既不是纯 AR 也不是纯 MA，无法用截尾肉眼读阶，改用 **EACF（O 三角顶点）+ 信息准则网格**。
- **ARMA 定阶比 AR/MA 难**：公共根近似相消会让真阶被简约模型替代，需更多数据、并以 BIC 兜底偏向简约。
- **平稳看 AR、可逆看 MA**：两个约束各管一边，互不干扰。
- **残差白噪声 = 模型充分**；若仍有自相关，说明 p 或 q 不够。
- **ARMA 是 ARIMA 的特例**：当数据不平稳，先差分 d 次再套 ARMA，就是 **ARIMA(p,d,q)**。

## 6. 一句话总结

> **ARMA 模型 = AR（用过去的值）+ MA（用过去的误差），用最少的参数刻画既有惯性又有滞后冲击的序列。** 完整流程是：
> **先验平稳（ADF）→ 因 ACF/PACF 双双拖尾、改用 EACF（方法一）+ (p,q) AIC/BIC 网格（方法二）定阶 → 拟合参数 → 查残差是否白噪声 → 做样本外预测并跑赢 baseline。**
> 本 demo 从一个真值为 ARMA(2,2)、AR/MA 根分离的序列出发，EACF 三角顶点与 AIC/BIC 三法一致地把阶数定回了 (2,2)，参数估计接近真值，残差通过白噪声检验，预测优于朴素基准——AR demo 与 MA demo 合体后的一次教科书式闭环。

### 易错点提醒
1. **想用 ACF/PACF 肉眼定 ARMA 的阶**——它俩都拖尾，读不出来，必须上 **EACF** 或信息准则网格。
2. **忽视公共根**——AR 与 MA 根相近会相消，真阶被简约模型替代，定阶失真。
3. **数据太少就定 ARMA**——区分真阶和简约近似需要更多样本（本 demo 用 900）。
4. **比较 IC 时样本量/收敛不一致**——同一训练集拟合，不收敛的组合跳过。
5. **混淆平稳与可逆**——ARMA 的平稳性看 AR 部分、可逆性看 MA 部分，别张冠李戴。
6. **不看残差就信预测**——残差有自相关说明 p 或 q 不够，或该升级到 ARIMA。
```

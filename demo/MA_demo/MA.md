# MA 模型学习总结

> 配套 demo：[ma_demo.py](ma_demo.py) —— 一条龙完整流程（造数据 → 平稳性检验 → 自动定阶 → 拟合 → 残差诊断 → 预测）。
> 本文把 MA 模型的核心理论和 demo 的每一步对应起来，并附上本次真实运行的数字。

---

## 1. MA 模型是什么

**MA(q)（q 阶移动平均模型）**：当前值由一个常数（均值）加上当前及过去 q 个**白噪声扰动**的线性组合构成。

$$
y_t = \mu + \varepsilon_t + \theta_1 \varepsilon_{t-1} + \theta_2 \varepsilon_{t-2} + \dots + \theta_q \varepsilon_{t-q},\quad \varepsilon_t \sim \text{白噪声}(0,\sigma^2)
$$

- **核心思想**：用「过去的随机冲击（误差）」而非「过去的观测值」来解释当下。冲击只在 **q 期内**有记忆，过了 q 期就彻底消失——这是 MA 与 AR 最本质的区别。
- **AR vs MA 一句话**：AR 是「对自己的过去值回归」，MA 是「对自己的过去误差回归」。
- 本 demo 真值：**MA(2)**，$y_t = 10.0 + \varepsilon_t + 0.6\,\varepsilon_{t-1} + 0.3\,\varepsilon_{t-2}$，$\sigma=1.0$，均值 $\mu=10.0$。

### 平稳性与可逆性

- **MA 过程恒平稳**（不需要任何条件）：它是有限个白噪声的线性组合，均值/方差/自协方差都不随时间变化。所以 ADF 检验对 MA 必然通过——步骤 1 只是走个流程确认。
- 真正要操心的是**可逆性（invertibility）**，它保证模型能唯一地用过去观测表达、参数可辨识。MA(2) 可逆条件：
  $$\theta_2+\theta_1<1,\qquad \theta_2-\theta_1<1,\qquad |\theta_2|<1$$
  demo 取 $\theta_1=0.6,\theta_2=0.3$，三条都满足 → 可逆。

### MA(q) 的理论自相关（定阶的理论基石）

MA(q) 的自相关函数 **在 lag q 之后恒为 0**（截尾）。本 demo 的理论值：

$$
\rho_1=\frac{\theta_1+\theta_1\theta_2}{1+\theta_1^2+\theta_2^2}=0.538,\quad
\rho_2=\frac{\theta_2}{1+\theta_1^2+\theta_2^2}=0.207,\quad
\rho_k=0\ (k>2)
$$

正是「ACF 在 lag 2 后截断」这一特征让我们能从数据反推出 q=2。

## 3. 各步骤详解 + 本次运行结果

### 步骤 0 · 数据生成
用已知参数的 MA(2) 递推生成 500 个点，前 300 个为**烧入期（burn-in）丢弃**。留最后 `h=50` 个点做**样本外测试**，其余 450 个做训练。建模时「假装不知道阶数」，最后再揭晓真值验证。

### 步骤 1 · 平稳性检验（ADF）
**ADF 检验**：$H_0$ = 「有单位根（不平稳）」，p < 0.05 → 平稳。（检验原理——$\delta=\rho-1$ 回归、为何单位根=不平稳——见 [AR.md 步骤1](../AR_demo/AR.md)；MA 恒平稳，故此处不展开。）

- 本次结果：**ADF = −12.023，p = 2.99e−22 ≪ 0.05 → 平稳，可建模 ✅**（MA 过程理论上恒平稳，这是预期内的结果）。
- 配图 [ma_fig1_identify.png](ma_fig1_identify.png)：序列图 + ACF + PACF。

### 步骤 2 · 定阶（本 demo 的精华：三法交叉验证）

定阶 = 确定 q 该取几。demo 用三种方法互相印证，最后投票：

> **先定搜索上限 `MAXLAG`**：三法都只在 $[1,\text{MAXLAG}]$ 内找 q，所以得先设个上限。黑盒下**不知道真实阶、不能拿真值反推**，于是用 **Schwert 规则** $\text{MAXLAG}=\lfloor 12\,(T/100)^{1/4}\rfloor$——**只依赖样本量 $T$**（本例 $T=450\Rightarrow$ **MAXLAG=17**，[ma_demo.py:83](ma_demo.py#L83)）。原则**宁大勿小**：它只是上界，真正定阶交给下面三法。**事后自检**：若定出的 q 顶到天花板，说明上限设小了、要调大重跑；本例 q=2 ≪ 17，绰绰有余 ✅。

**2a · ACF 截尾法** —— MA 模型的识别口诀（与 AR 正好相反）：

| | ACF（自相关） | PACF（偏自相关） |
|---|---|---|
| AR(p) | 拖尾（tails off） | **p 阶后截尾**（cuts off） |
| **MA(q)** | **q 阶后截尾**（cuts off） | 拖尾（tails off） |
| ARMA | 拖尾 | 拖尾 |

→ 看 ACF 在第几阶之后掉进置信带，那个阶就是 q。
⚠️ **关键工程细节**：MA 定阶必须用 **Bartlett 公式**置信带（[ma_demo.py:88-97](ma_demo.py#L88-L97)），它在 lag>q 后**逐步变宽**：$\text{SE}_k=\sqrt{(1+2\sum_{j<k}\rho_j^2)/N}$。若图省事用平直的 $\pm1.96/\sqrt N$，远端噪声极易冒出**伪显著 lag**（本 demo 实测平直带会误报 lag 9）。本次用 Bartlett 带：**连续显著到 lag = 2**。

**2b · AIC / BIC 信息准则网格** —— 逐阶拟合 `ARIMA(0,0,q)`，选准则**最小**的阶。
- **是什么**：都是「$-2\ln L+\text{复杂度罚}$」的打分，**越小越好**——第一项要似然 $L$ 大（拟合好），第二项罚参数多（防过拟合），两者权衡。
- 公式只差**惩罚项**：$\text{AIC}=-2\ln L+2k$、$\text{BIC}=-2\ln L+k\ln n$（$k$=参数个数、$n$=样本量）；$n>7$ 起 $\ln n>2$，故 BIC 罚得更重、更「吝啬」，倾向选更小的阶。
- 本次：AIC 最小 @ q=2，BIC 最小 @ q=2（见下表，q=2 处见底后单调回升）。

```
  q        AIC        BIC
  1    1328.77    1341.10
  2    1296.81    1313.24   <- 双双最小
  3    1298.77    1319.32
  ...    (此后逐阶回升)
```

**2c · 过拟合系数显著性法** —— 故意拟合到搜索上限 MA(MAXLAG)（本例 MA(17)），看哪些滞后项系数显著（p<0.05），不显著的高阶项应剔除。本次：显著的最高滞后 = 2。

**投票汇总**：`ACF=2 | AIC=2 | BIC=2 | 显著性=2` → **四法一致，最终定阶 q = 2（以 BIC 为准）**。
配图 [ma_fig2_order.png](ma_fig2_order.png)：AIC/BIC 随阶数变化，都在 q=2 处见底。

### 步骤 3 · 拟合
**怎么拟合的（和 AR 的根本区别）**：MA 的回归元是**过去的误差 $\varepsilon_{t-1},\varepsilon_{t-2}$，它们不可观测**，没法像 AR 那样把滞后项摆成设计矩阵直接跑 OLS。于是改用**极大似然 + 状态空间（卡尔曼滤波）**：把 MA(2) 写成状态空间形式，卡尔曼滤波在给定参数下递推出每步的 $\hat\varepsilon_t$ 和似然，优化器再迭代调 $(\mu,\theta_1,\theta_2,\sigma^2)$ 让似然最大。代码 `ARIMA(train, order=(0,0,q_hat), trend="c").fit()`（[ma_demo.py:143](ma_demo.py#L143)）。**这正是 MA 比 AR 慢的根本原因**：AR 一步 OLS 出解析解，MA 要反复迭代优化。

估计参数 vs 真值：

| 参数 | 估计值 | 真值 |
|---|---|---|
| const (μ) | +9.911 | 10.0 |
| θ₁ (ma.L1) | +0.665 | 0.6 |
| θ₂ (ma.L2) | +0.284 | 0.3 |
| σ² (sigma2) | +1.025 | 1.0 |

估计值都很接近真值 → 模型抓对了结构。

### 步骤 4 · 残差诊断（判断模型是否充分）
**好模型的残差应近似白噪声**。

- **Ljung–Box 检验**：$H_0$ = 「残差无自相关」。p > 0.05 → 残差是白噪声 ✅。
  本次：**Ljung-Box(10) p = 0.963 ≫ 0.05 → 残差无自相关，模型充分**。
- 残差均值 ≈ −0.001，标准差 ≈ 1.015（贴近真值 σ=1.0）。
- 配图 [ma_fig3_residuals.png](ma_fig3_residuals.png)：残差时序图、残差 ACF（全在带内）、直方图（近正态）、Q-Q 图（贴直线）。

### 步骤 5 · 样本外预测 + 评估
对未参与训练的最后 50 个点预测并评估：

- **RMSE = 1.359，MAE = 1.151**
- **朴素基准（last-value）RMSE = 1.859** → **MA 更优 ✅**
- 配图 [ma_fig4_forecast.png](ma_fig4_forecast.png)：可清楚看到 MA 的**有限记忆**——**只有前 q=2 步**带有信息（误差还没耗尽），从第 3 步起预测**立刻变成一条平直线 = 均值 μ=10**，置信区间也在 q 步后变为恒定宽度。这与 AR「逐渐收敛到均值」的平滑回复形成鲜明对比。

## 4. 关键概念速查（AR ↔ MA 对照）

| 维度 | AR(p) | MA(q) |
|---|---|---|
| 回归对象 | 过去的**观测值** $y_{t-k}$ | 过去的**误差** $\varepsilon_{t-k}$ |
| 定阶看 | **PACF 截尾** @ p | **ACF 截尾** @ q |
| 平稳性 | 需满足条件（否则差分） | **恒平稳** |
| 另需关注 | 平稳条件 | **可逆条件** |
| 估计方法 | 最小二乘（快） | 极大似然/状态空间（慢） |
| 多步预测 | 平滑**收敛**到 μ | 超过 q 步**立刻**变平到 μ |
| 记忆长度 | 理论无限（指数衰减） | 严格有限（q 期） |

- **ACF 截尾 + PACF 拖尾 → 认 MA；截尾的阶数 = q。** 这是肉眼定阶的第一直觉。
- **MA 定阶务必用 Bartlett 带**：平直带在远端会误报伪显著 lag（本 demo 实测会误报 lag 9）。
- **AIC vs BIC**：都越小越好；BIC 惩罚更重、更偏好简约模型。两者矛盾时以 BIC 为准。
- **残差白噪声 = 模型充分**的判据；若残差仍有自相关，说明阶数不够或该上 ARMA。
- **MA 的有限记忆**：长期预测毫无信息量（直接吐均值），所以 MA 同样只适合**极短期**预测。

## 6. 一句话总结

> **MA 模型 = 用过去的随机冲击（误差）解释当下。** 完整流程是：
> **确认平稳（ADF，MA 必过）→ 用 ACF（Bartlett 带）+ AIC/BIC + 系数显著性定阶 → 拟合参数 → 查残差是否白噪声 → 做样本外预测并跑赢 baseline。**
> 本 demo 从一个真值为 MA(2) 的序列出发，四法一致地把阶数定回了 2，参数估计接近真值，残差通过白噪声检验，预测优于朴素基准——与 AR demo 互为镜像的一次教科书式闭环。

### 易错点提醒
1. **把 MA 当 AR 定阶**——AR 看 PACF 截尾，MA 看 **ACF 截尾**，看错对象全盘皆错。
2. **MA 用平直置信带**——远端噪声会冒出伪显著 lag，务必用 **Bartlett 带**。
3. **担心 MA 不平稳**——MA 恒平稳，真正要查的是**可逆性**。
4. **不看残差就信预测**——残差有自相关说明阶数不够，或该升级到 ARMA。
5. **拿 MA 做多步预测**——超过 q 步就是一条均值平直线，远期零信息。
